Standard matchers

Matchers take a regular expression pattern
fileext: jpg Matches the naive file extension
frameurl: http://www.google.com Matches the URL of the current frame
linktext: Click here to download Matches the link text or element title attribute
mediatype: image Matches the media type: audio, video, or image
naivefilename: cat.jpg Matches the naive filename
pagetitle: Welcome to my page! Matches the current page title
pageurl: http://www.google.com Matches the current page URL
selectiontext: "X-Cops" is the twelfth Matches the selection text
srcurl: cdn.xyz.com Matches the source URL for media

Filename rewrite matchers

Usable only in filename rewrite rules
filename: actual_filename.jpg Matches the final filename

Optionally, set up rules to route downloads or rewrite filenames. The first matching rule applies. Separate multiple rules with an empty line. Each rule must have an into clause, at least one matcher clause, and an optional capture clause. Multiple variables can be used in the into clause. 

(matchername): regex
into: directory pattern

CHROME USERS: Special characters in the filename (eg. : or /) are pre-replaced with the character - by Chrome. Match against - instead of these special characters. For example, use (.*)-large to match filenames instead of (.*):large.
